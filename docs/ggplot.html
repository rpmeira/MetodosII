Introduo

O ggplot um pacote de visualizao de dados e sua documentao pode ser encontrada em seu site oficial. Para instal-lo, bastar rodar o comando (no acrescentar no documento .Rmd, mas rodar no console):

install.packages("ggplot2")

E para utiliz-lo, basta rodar a funo library() (no caso de arquivos .Rmd, acrescentar no incio do documento). uma boa prtica rodar no incio do arquivo todos os pacotes que iremos utilizar:

library(ggplot2)
library(dplyr)
library(sidrar)
library(lubridate)
library(rmarkdown)

O ggplot prope uma forma de criar grficos baseada camadas. Com isso, possvel adicionar, alterar e/ou remover camadas especficas do grfico, o que permite um alto nvel de customizao. Em geral, um grfico do ggplot apresenta as seguintes camadas:

  • Dados
  • Mapeamento
  • Geometrias
  • Rtulos
  • Escalas
  • Temas

Porm, apenas as trs primeiras so essenciais e necessrias de ser includas em qualquer grfico.

Para facilitar, vamos utilizar os dado da seo sobre o pacote dplyr:

ipca <- get_sidra(api = '/t/1419/n1/all/v/63,66/p/all/c315/7170,7445,7486,7558,7625,7660,7712,7766,7786/d/v63%202,v66%204')

ipca_valor <- ipca %>%
  filter(`Varivel` == "IPCA - Variao mensal") %>%
  select("Data" = `Ms`, "Grupo" = 13, "Valor") %>%
  mutate("Data" = `Data` %>% mdy()) %>%
  mutate("Valor" = `Valor` / 100) 

ipca_valor %>%
  paged_table()

Para criar grficos com o pacote ggplot precisamos fornecer pelo menos camadas referentes aos dados, mapeamento e geometria. Os dados (no caso, ipca_valor) entram como argumento da funo ggplot(); o mapeamento fornecido nos parmetros da funo aes(); e a geometria indicada pela funo correspondente geometria desejada, que no exemplo a seguir ser uma linha, geom_line().

Cada camada adicionada por meio do sinal de somatrio, +, que funciona de forma semelhante ao pipe (%>%).

Vamos adicionar essas camadas uma por uma e ver que o grfico s surge quando inclumos as trs camadas essenciais.

ipca_valor %>%
  ggplot() 


ipca_valor %>%
  ggplot() +
  aes(x = `Data`, y = `Valor`, color = `Grupo`)


ipca_valor %>%
  ggplot() +
  aes(x = `Data`, y = `Valor`, color = `Grupo`) +
  geom_point()

Mapeamento

A funo aes() faz o mapeamento dos dados, ou seja, ela atribui variveis (presentes nos dados que estamos utilizando) a aspectos do grfico, como o eixo x, o eixo y, a cor ou o tamanho. Ela pode receber um, dois, trs, ou mais parmetros, dependendo do objetivo. No exemplo acima, utilizamos trs parmetros: um para o eixo x, um para o eixo y e um para as cores. Porm, poderamos ter utilizado menos ou mais mapeamentos.

Eixos

No exemplo acima, utilizamos as Datas no eixo x e o Valor no eixo y, mas poderamos, por exemplo, mapear a varivel Valor no eixo x e a varivel Grupo no eixo y:

ipca_valor %>%
  ggplot() +
  aes(x = `Valor`, y = `Grupo`) +
  geom_point()

Cor

Adicionalmente, podemos mapear a varivel Data no parmetro color (dependendo do tipo de geometria, utilizamos o parmetro fill):

ipca_valor %>%
  ggplot() +
  aes(x = `Valor`, y = `Grupo`, color = `Data`) +
  geom_point()

Podemos tambm repetir uma varivel j utilizada em outro mapeamento. Por exemplo, alm de mapear o Grupo no eixo y, vamos mape-lo tambm na cor:

ipca_valor %>%
  ggplot() +
  aes(x = `Valor`, y = `Grupo`, color = `Grupo`) +
  geom_point()

Tamanho

Outro mapeamento muito utilizado o tamanho (por meio do parmetro size). Assim como a cor, ele ajuda na distino de diferentes grupos de dados (para facilitar a visualizao, filtramos apenas um dos grupos da tabela):

ipca_valor %>%
  filter(`Grupo` == "2.Habitao") %>%
  ggplot() +
  aes(x = `Data`, y = `Valor`, size = `Valor`) +
  geom_point()

Geometria

Ao invs de um grfico de pontos, poderamos ter um grfico de linha, barra, ou outro qualquer. Porm, necessrio adequar a geometria com o tipo de dados que se est trabalhando. Alguns tipos de geometria requerem mapeamentos especficos.

Vejamos alguns exemplos:

rea

ipca_valor %>%
  ggplot() +
  aes(x = `Data`, y = `Valor`, color = `Grupo`) +
  geom_area()

Podemos mapear o Grupo com cores de preenchimento ao invs do contorno, utilizando o parmetro fill, ao invs de color:

ipca_valor %>%
  ggplot() +
  aes(x = `Data`, y = `Valor`, fill = `Grupo`) +
  geom_area()

Boxplot

ipca_valor %>%
  ggplot() +
  aes(x = `Valor`, y = `Grupo`) +
  geom_boxplot()

Coluna

ipca_valor %>%
  ggplot() +
  aes(x = `Data`, y = `Valor`, fill = `Grupo`) +
  geom_col()

Densidade

ipca_valor %>%
  ggplot() +
  aes(x = `Valor`, color = `Grupo`) +
  geom_density()

Linha

ipca_valor %>%
  ggplot() +
  aes(x = `Data`, y = `Valor`, color = `Grupo`) +
  geom_line()

Cada funo de geometria dessas possui parmetros especficos para mudar alguns detalhes da sua forma.

possvel acrescentar mais de uma geometria. Vamos pegar o exemplo acima e acrescentar pontos (com a funo geom_point()), uma reta horizontal (com a funo geom_hline()) e uma reta vertical (com a funo geom_vline()).

ipca_valor %>%
  ggplot() +
  aes(x = `Data`, y = `Valor`, color = `Grupo`) +
  geom_line() +
  geom_point() +
  geom_hline(yintercept = 0.06) +
  geom_vline(xintercept = "2015-07-20" %>% ymd())

Rtulos

Os rtulos de um grfico abrangem desde o ttulo e o subttulo at a legenda e os nomes dos eixos. No ggplot, esses rtulos so indicados na funo labs(). Alguns desses rtulos j aparecem de forma automtica nos grficos bases, porm possvel alter-los.

Vamos pegar o grfico do exemplo acima e adicionar/modificar alguns rtulos:

ipca_valor %>%
  ggplot() +
  aes(x = `Data`, y = `Valor`, color = `Grupo`) +
  geom_line() +
  labs(
    title = "Inflao mensal por grupos",
    subtitle = "Evoluo no tempo",
    caption = "Fonte: Elaborao prpria. Dados do SIDRA",
    x = "Ano",
    y = "Inflao"
  )

Esse rtulos podem ser customizados depois, alterando-se as propriedades do tema do grfico, como veremos mais a frente.

Escalas

Frequentemente desejamos mudar as escalas dos eixos tanto x quanto y, para facilitar o entendimento dos dados. Caso queiramos dar um zoom no grfico, prefervel manipular os dados com o pacote dplyr. As escalas aqui so mais teis para manipular os eixos.

Existem diversas funes para manipular as escalas, cada um adequada para um tipo especfico de de grfico Usualmente essas funo so da forma scale_x_... e scale_y_.... Nessas funes, ns passamos parmetros que modificam a escala.

Vamos pegar o mesmo grfico base do exemplo anterior e modificar as escalas dos eixos. Vamos modificar a escala das datas com a funo scale_x_date() e a escala dos valores com a funo scale_y_continuous()

ipca_valor %>%
  ggplot() +
  aes(x = `Data`, y = `Valor`, color = `Grupo`) +
  geom_line() +
  scale_x_date(date_breaks = "1 year", 
               date_labels = "%Y") +
  scale_y_continuous(breaks = seq(from = -0.03, to = 0.06, by = 0.01),
                     labels = scales::percent_format(accuracy = 1))

Porm, nem todos os aspectos do grfico podem ser modificados pela funo theme(). No exemplo acima, tivemos que chamar a funo guides() para modificar a estrutura da legenda.

possvel salvar as especificaes de tema para que essas sejam utilizadas em todos os grficos seguintes, por meio das funes theme_set() (que define um tema) e theme_update() que modifica os elemtentos desse tema:

theme_set(theme_hc())

theme_update(
